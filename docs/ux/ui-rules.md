# Anacronia UI Rules

## Status

- Role: current UI hygiene rules.
- Subordinate to the UX contracts, app shell decisions, and shadcn/Rhea system.
- Read when reviewing visible product UI or repeated fact placement.

## No Duplicate Facts

A user-facing fact should appear once in a view. Do not repeat the same metadata in the header, body cards, tabs, and side panels.

Actions may refer to the same entity, such as `Open provider record`, but they should not restate metadata already shown elsewhere.

If information feels important enough to repeat, promote it to the one correct location instead.
