# Preserve full Provider records for imported material

When an online Provider object contributes usable local material to a Collection, Anacronia should preserve the complete provider object record locally. The raw provider record is the source canon for imported online material: normalized Museum Object fields, Image Asset fields, Descriptors, Match records, rights display, exports, and detail UI projections are derived views over that source record.

This refines ADR-0010 by defining the preservation scope: complete object/detail records for imported online Provider material are source canon; transient search/list responses and skipped candidates are not automatically part of that canon.

**Status:** accepted

**Considered Options:** Keeping only normalized fields would keep storage smaller and simplify schema design, but it would make early information-design choices irreversible and would hide provider-specific meaning. Preserving every fetched search candidate or every search/list response would maximize auditability, but it would also expand storage and operational scope beyond the material the user actually added to a Collection.

**Consequences:** Provider adapters must preserve complete object/detail metadata payloads for online Provider material when at least one Image Asset completes `standard-1024` and `thumb-256` validation. If an object has no successfully imported Image Assets, or is rejected by a provider-specific eligibility gate, it may be recorded as a skipped candidate without becoming part of the raw-record source canon. Separate image/detail or IIIF metadata payloads, provider response wrappers, and search-result fields should be retained when they affect import eligibility, rights, warnings, matches, display, or future remapping. Full-resolution image binaries are not provider records; ADR-0005 continues to govern local image derivatives. Raw provider data should be written atomically with, or before, normalized imported rows are accepted. Stored provenance should include Provider ID, source object ID, raw record path, fetched time, and provider version, timestamp, or hash when available. `Remove from Collection` must not delete imported raw provider canon. Global delete, cleanup, and migrations need explicit policy and must distinguish raw provider source records from temporary provider responses, skipped candidates, imported Image Assets, and local derivatives. Normalized tables remain derived views over imported provider source data, not the only truth.
